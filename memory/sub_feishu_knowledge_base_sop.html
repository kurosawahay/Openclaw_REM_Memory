飛書知識庫結構化整理 SOP

aliases: 飛書知識庫, Lark知識庫, 知識庫整理, 結構化整理, Feishu知識庫

📅 記錄時間

2026-03-09 (Asia/Taipei)

🎯 任務目標

將 OpenClaw 的開發研究經驗與技能知識系統化整理到飛書知識庫，建立結構化的知識管理系統。

🔧 使用工具

  • 飛書文檔 API：feishu_doc 工具
  • 飛書知識庫 API：feishu_wiki 工具
  • OpenClaw 記憶系統：memory_search、memory_get、memory/ 目錄

📋 完整 SOP 步驟

步驟 1：分析現有知識結構

  1. 盤點現有技能：列出所有已學習的技能模組（20+ 個）
  2. 分類整理：將技能分為 7 大類別：
    • 🏗️ 核心系統與開發
    • 🧠 知識管理與記憶系統
    • 🌐 網頁研究與自動化
    • 🏢 飛書/Lark 深度整合
    • 🛡️ 安全、維護與檔案系統
    • 💬 通訊、協作與多媒體
    • 📱 硬體節點與介面控制

步驟 2：設計知識庫架構

  1. 章節式結構：設計 0-9 章節編號體系

    • 0. 知識庫首頁（導航目錄）
      1. 系統概覽與架構
      1. 核心系統模組
      1. 記憶與資料系統
      1. Skills 技術系統
      1. AI 能力模組
      1. 自動化與代理系統
      1. 實戰應用案例
      1. 第三方平台整合
      1. 開發指南

  2. 扁平化設計：最多三層結構，避免過度巢狀

  3. 統一格式：每個分類頁面採用「介紹、設定、使用、範例、FAQ」五段式標準結構

步驟 3：創建知識庫文檔

  1. 獲取文檔 token：透過 feishu_wiki 或直接從 URL 提取
  2. 創建新文檔：使用 feishu_doc 的 create 動作
  3. 設定權限：務必設定 grant_to_requester: true，讓黑澤大人擁有編輯權限
  4. 文檔命名規範：使用「章節編號 + 章節名稱」格式

步驟 4：內容整理與撰寫

  1. 從記憶系統提取：使用 memory_search 搜尋相關技能記錄
  2. 結構化整理：
    • 介紹：功能概述、適用場景
    • 設定：安裝配置步驟、參數說明
    • 使用：基本用法、進階功能
    • 範例：實際應用案例、程式碼片段
    • FAQ：常見問題與解決方案
  3. 超連結建立：使用 docx:文檔ID 格式建立文檔間連結
  4. 日期標註：所有技術資料標註來源日期，確保時效性

步驟 5：開發日誌系統建立

  1. 主目錄文檔：創建開發日誌系統目錄
  2. 每日獨立節點：為每個開發日創建獨立文檔
  3. 超連結網絡：建立完整的導航系統
  4. 時間排序：最新日誌在最上方，歷史日誌往下排列
  5. 統一格式：每個日誌包含「當天總結」、「詳細報告」、「錯誤與解決方案」

步驟 6：APIYI Flux 整合 SOP

  1. API 研究：

    • 確認 APIYI 支援的圖片生成模型
    • 確認 API 端點格式（OpenAI 相容格式）
    • 確認 API Key 格式與權限

  2. OpenClaw 配置：

  3. 測試驗證：

    • 使用 flux-dev 進行基本連線測試
    • 測試圖片生成功能
    • 驗證 API 回應格式

  4. 技術文檔撰寫：

    • 完整設置教程
    • Agent 使用指南
    • 錯誤處理與注意事項

步驟 7：知識庫同步與維護

  1. 定期更新：每月更新 AI 大模型價格比較表
  2. 內容驗證：確保知識庫內容與實際系統配置一致
  3. 權限管理：定期檢查文檔權限設定
  4. 備份機制：本地記憶系統與飛書知識庫雙向同步

🎯 關鍵技術點

1. 飛書文檔權限管理

  • 問題：機器人創建的文檔預設只有機器人權限
  • 解決方案：創建文檔時設定 grant_to_requester: true
  • 最佳實踐：所有文檔都要明確授予使用者編輯權限

2. 知識庫節點訪問

  • 問題：無法直接瀏覽知識庫節點清單（feishu_wiki nodes 回傳 400 錯誤）
  • 解決方案：透過直接提供節點 token 讀取和編輯文檔
  • 替代方案：建立文檔索引，手動管理連結

3. 結構化內容設計

  • 原則：每個功能只出現在一個分類中，避免內容重複
  • 格式：統一使用 Markdown 格式，包含標題、列表、程式碼區塊
  • 連結：建立完整的超連結網絡，方便導航

4. 價格資訊管理

  • 時效性：所有價格資料標註來源日期
  • 定期更新：建議每月更新一次
  • 場景建議：根據不同使用場景提供模型選擇建議
