Cronjob 分散管理系統

aliases: Cron任務, 定時任務, 任務管理, 分散式管理

建立時間

2026-02-23 11:56 (Asia/Taipei)

問題背景

四個代理人（雷姆、萬、蒼影、鼬）的 cronjob 都寫在同一份 jobs.json 檔案中，容易造成衝突和維護困難。

解決方案

建立分散式 cronjob 管理系統，讓每個代理人管理自己的檔案，透過自動合併機制統一載入。

系統架構

檔案結構

~/.openclaw/cron/
├── jobs.json                  # 主合併檔案 (自動產生)
├── merge_jobs.sh              # 合併腳本
├── backups/                   # 自動備份
│   └── jobs.json.backup.*
└── split/                     # 分割檔案目錄
    ├── main.json              # 雷姆的 cronjob (main agent)
    ├── xact_mission_manager.json  # 萬的 cronjob
    ├── xact_searching.json    # 蒼影的 cronjob
    └── xact_creative_manager.json # 鼬的 cronjob

工作流程

  1. 分散管理：每個代理人只修改自己的分割檔案
  2. 自動合併：每小時執行合併腳本，整合所有分割檔案
  3. 備份保護：每次合併前自動備份原檔案
  4. 系統載入：OpenClaw 載入合併後的 jobs.json

實施步驟

1. 分割現有 cronjob

將原有的 jobs.json 按 agentId 分割為四個檔案：

  • main.json (雷姆)：1 個任務（每日記憶揉捏）
  • xact_mission_manager.json (萬)：7 個任務
  • xact_searching.json (蒼影)：3 個任務
  • xact_creative_manager.json (鼬)：1 個任務

2. 建立合併腳本

建立 merge_jobs.sh，功能包括：

  • 讀取所有分割檔案
  • 合併為單一 jobs.json
  • 自動備份原檔案
  • JSON 格式驗證
  • 任務統計輸出

3. 新增合併 cronjob

建立新的 cronjob 任務：

  • 名稱：合併所有 cronjob 檔案
  • 代理人：main (雷姆)
  • 排程：每小時執行一次 (0 * * * *)
  • 時區：Asia/Taipei
  • 動作：執行合併腳本

4. 建立說明文件

README.md 包含：

  • 系統架構說明
  • 操作指南
  • 備份與恢復方法
  • 注意事項

技術細節

合併腳本功能

檔案路徑記錄

  • 合併腳本: ~/.openclaw/cron/merge_jobs.sh
  • 分割目錄: ~/.openclaw/cron/split/
  • 備份目錄: ~/.openclaw/cron/backups/
  • 說明文件: ~/.openclaw/cron/README.md

任務分佈 (分割後)

雷姆 (main)

  • 每日記憶揉捏 (雷姆) - 22:00
  • 合併所有 cronjob 檔案 - 每小時

萬 (xact_mission_manager)

  • Line群組每日分析 (disabled)
  • 伺服器每小時檢查 (disabled)
  • 早上：輸出「今日總覽」 - 07:00
  • 午間：只輸出「三件關鍵推進」+「等待/卡關超時提醒」 - 13:00
  • 晚上：日結 - 22:30
  • 每日：記憶揉捏 (萬) - 00:00
  • 測試 cron 連線 - 每 5 分鐘

蒼影 (xact_searching)

  • 每日情報蒐集 - 00:00
  • 趨勢報告生成 - 14:00 (一、三、五、日)
  • 記憶揉捏 (蒼影) - 15:00

鼬 (xact_creative_manager)

  • 每日記憶揉捏 (鼬) - 07:00

操作規範

新增/修改 cronjob

  1. 找到對應的代理人分割檔案
  2. 編輯該檔案（保持 JSON 格式）
  3. 等待合併腳本自動執行（每小時）
  4. 或手動執行 ./merge_jobs.sh

優點與效益

解決的問題

  1. 零衝突：每個代理人獨立檔案，不會互相干擾
  2. 權責分明：誰的任務誰負責維護
  3. 版本控制：可個別追蹤修改歷史
  4. 錯誤隔離：單一檔案錯誤不影響其他任務

系統效益

  1. 自動化：每小時自動合併，確保同步
  2. 備份保護：每次合併前自動備份
  3. 易於維護：分散管理，集中載入
  4. 可擴展：未來新增代理人只需新增分割檔案

注意事項

  1. 不要直接編輯 jobs.json：此檔案由合併腳本自動產生
  2. Gateway 重啟：重大修改後建議重啟 Gateway
  3. 格式驗證：合併腳本會檢查 JSON 格式
  4. 錯誤處理：合併失敗時保留原檔案，避免系統中斷

後續維護

定期檢查

  • 每月檢查合併腳本執行記錄
  • 確認備份檔案正常
  • 檢查各代理人任務狀態

擴展性

  • 新增代理人時，只需在 split/ 目錄新增對應檔案
  • 合併腳本會自動包含新檔案

故障排除

  1. 合併失敗：檢查分割檔案 JSON 格式
  2. 任務不執行：檢查 Gateway 服務狀態
  3. 備份遺失：手動從最近備份恢復
