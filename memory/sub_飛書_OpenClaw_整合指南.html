🎯 核心目標

讓 AI Agent 直接操作飛書文檔——讀取內容、寫入內容、管理知識庫、操作多維表格。不是「AI 幫你總結飛書文檔」，而是 AI 自己登入飛書，直接讀寫你的文檔。

🔧 前提條件

  1. OpenClaw 已安裝並運行（版本 >= 0.3.x）
  2. 一個飛書組織（個人版或企業版都行）
  3. 飛書開放平台應用（下面教你創建）

🎯 實戰場景：我們怎麼用的

場景 1：AI 自動寫日報

我們每天讓 AI 把當天的工作成果寫進飛書文檔。AI 讀取 Git log、伺服器狀態、發布記錄，自動生成日報並寫入指定的飛書文檔。

使用者只需要在飛書群裡說一句「寫今天的日報」，AI 就會：

  1. 收集當天資訊（Git 提交、任務完成情況、伺服器狀態）
  2. 生成 Markdown 格式的日報
  3. 用 feishu_doc 的 write 動作寫入指定文檔
  4. 在群裡回覆「日報已更新」並附上文檔連結

場景 2：知識庫自動整理

AI 定期掃描知識庫結構，把沒分類的文檔移到對應目錄，把過期文檔歸檔。

場景 3：多維表格任務看板

把專案任務放在飛書多維表格裡，AI 可以：

  • 自動創建新任務（從 GitHub Issues 同步）
  • 更新任務狀態（程式碼合併後自動標記完成）
  • 每天發任務摘要到群裡

🧪 驗證方法

配置完成後，在飛書裡 @你的機器人，發送以下訊息測試：

測試 1 — 讀文檔：

幫我讀一下這個文檔：https://xxx.feishu.cn/docx/你的文檔token

預期：AI 返回文檔標題和內容摘要。

測試 2 — 寫文檔：

在這個文檔裡追加一行「測試寫入 - 當前時間」

預期：打開飛書文檔能看到新增的內容。

測試 3 — 知識庫瀏覽：

列出我們的知識庫

預期：AI 返回所有可訪問的知識空間列表。

測試 4 — 多維表格：

幫我在 https://xxx.feishu.cn/base/XXX?table=YYY 裡新增一條記錄

預期：打開多維表格能看到新增的行。

❓ 常見問題

Q：提示「沒有權限」怎麼辦？

A：檢查兩個地方：

  1. 飛書應用的權限是否都開了（權限管理頁面）
  2. 目標文檔/知識庫是否對機器人開放了訪問權限。知識庫需要把機器人加為成員。

Q：機器人創建的文檔我打不開？

A：創建文檔時要傳 owner_open_id 參數（使用者的 open_id），這樣使用者會自動獲得文檔的 full_access 權限。否則只有機器人自己能訪問。

Q：Markdown 表格寫不進去？

A：飛書文檔的 Markdown 寫入不支援表格語法。要創建表格，用 create_table_with_values 動作：

Q：文檔很大，讀取超時？

A：先用 read 獲取概覽，然後用 list_blocks 獲取完整塊結構。對於只需要部分內容的場景，可以用 get_block 精確讀取特定塊。

Q：機器人沒有根目錄，創建資料夾報錯？

A：飛書機器人沒有「我的空間」。需要使用者先手動創建一個資料夾，分享給機器人後，機器人才能在裡面創建子資料夾和文檔。

📖 重要提醒（雷姆的經驗總結）

  1. 權限設定：創建文檔時務必使用 grant_to_requester: true 或傳入 owner_open_id
  2. 知識庫成員：機器人需要被加入知識庫空間才能創建新節點
  3. 國際版差異：Lark 國際版與飛書中國版在 API 端點和功能上略有差異
  4. 錯誤處理：遇到 403 權限錯誤時，應創建新文檔並建議使用者手動移動
