2026-02-21 NotebookLM 技能安裝與測試

aliases: 2026-02-21 NotebookLM 技能安裝與測試

📍 背景

根據「一口新饭」的文章《為什麼你的 OpenClaw 一定要安裝 NotebookLM Skill》，進行技能安裝與測試。

📖 文章重點摘要

核心概念：AI 指揮 AI

  • 問題：NotebookLM 能力強大，但 source 管理（收集、導入、清洗、更新）是純手工的
  • 解決方案：OpenClaw + NotebookLM Skill = 自動化 source 管理
  • 效果：從「操作員」變成「指揮官」

實際案例展示

  • 目標：將 OpenClaw 官方文檔全站（524 頁）導入 NotebookLM
  • OpenClaw 自動完成：
    1. 抓取 sitemap，提取全部 URL
    2. 批量導入 524 個頁面
    3. 發現並清理重複（中文翻譯版 255 個 + URL 重複 38 個）
    4. 審計結果：269 個乾淨 source，0 missing，0 extra，0 duplicate
  • 時間：約 30 分鐘（完全自動，無需監控）

NotebookLM 的強大能力

  1. 研究與交互：Source-grounded 問答、深度研究、多 source 綜合
  2. 內容生成：播客、視頻教程、演示文稿、思維導圖、測驗、信息圖
  3. 應用場景：學習加速、內容生產、團隊協作、研究分析

🔧 安裝過程

步驟 1：訪問 GitHub 倉庫

步驟 2：安裝 notebooklm-py

步驟 3：Google 授權

授權流程：

  1. 瀏覽器自動開啟
  2. 黑澤大人手動登入 Google 帳號
  3. 進入 NotebookLM 首頁
  4. 按 ENTER 儲存授權

步驟 4：功能測試

⚠️ 遇到的問題

問題 1：RPC API 錯誤

23:36:09 ERROR [notebooklm._core] RPC GET_NOTEBOOK failed after 0.296s
Error: No result found for RPC ID: rLM1Ne

問題分析

  1. API 變更：Google 可能更新了內部 RPC 方法
  2. 技能過時：notebooklm-py 可能需要更新
  3. 暫時影響：核心問答功能暫時不可用

可用功能確認

  • ✅ 授權管理：notebooklm login, notebooklm status
  • ✅ 筆記本列表：notebooklm list (顯示 30+ 個筆記本)
  • ❌ 筆記本選擇：notebooklm use (RPC 錯誤)
  • ❌ 問答功能：notebooklm ask (RPC 錯誤)
