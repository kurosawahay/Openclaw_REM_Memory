Cronjob Telegram 發送機制重大故障

aliases: Cron任務, 定時任務, 任務管理, 分散式管理

故障發現時間

2026-02-24 10:45 (Asia/Taipei)

故障描述

所有 cronjob 顯示 cron announce delivery failed，但手動發送正常。

故障時間線

10:25 - 萬回報問題

cron announce delivery failed

10:30 - 初步檢查

  1. ✅ 格式檢查：所有任務格式正確
  2. ✅ accountId 檢查：每個任務有對應 bot
  3. ✅ 手動測試：openclaw message send 成功
  4. ❌ Cron 測試：仍然失敗

10:45 - 深入分析

日誌發現關鍵錯誤：

Error: Telegram bot token missing. Set TELEGRAM_BOT_TOKEN or channels.telegram.botToken.

11:13 - 系統重置

  • ✅ 完整備份：15個任務全部備份
  • ✅ 系統清空：重置 cronjob 系統
  • ✅ 公告發布：群組通知暫停所有定時任務

技術分析

已知事實

  1. ✅ 格式正確：telegram:[TG_GROUP_ID]
  2. ✅ accountId 正確：每個任務有對應 bot
  3. ✅ 手動發送成功：Message ID: 1225
  4. ✅ 預設 botToken 設定：已設定雷姆的 botToken
  5. ❌ Cron announce 失敗：顯示 token missing

根本問題推測

OpenClaw cron announce 機制 與 手動 message send 使用不同的路徑：

手動發送路徑（成功）

openclaw message send 
    → 使用 --account 參數
    → 讀取 channels.telegram.accounts
    → 成功發送

Cron announce 路徑（失敗）

cron announce
    → 可能只檢查 channels.telegram.botToken
    → 不讀取 accounts 設定
    → 顯示 token missing
    → 發送失敗

設定結構分析

解決嘗試

嘗試1：設定預設 botToken

結果：❌ 仍然失敗

嘗試2：檢查所有設定

發現：botToken 為空字串，但 accounts 有4個 bot

嘗試3：測試各 bot 手動發送

結果：✅ 全部成功

嘗試4：系統重置

  1. 完整備份：emergency_backup_20260224_111311
  2. 系統清空：jobs.json 重置
  3. 建立乾淨任務：無 delivery 的合併任務
