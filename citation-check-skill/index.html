📚 Citation Check Skill

Citation Check 引文驗證技能

檢測、驗證並強制執行 LLM 輸出中的引用來源

📋 總覽

核心理念：大多數工具在生成後添加引用。此工具將引用設為生成約束——如果無法引用，就不應該產生聲明。
🎯

輕量級模組

專為研究、簡報和代理流程設計的精簡引用驗證層

🔒

強制引用

驗證每個事實聲明是否有引用，並阻止缺失或虛構的引用

🔍

視覺支援

可讀取簡報、PDF、圖表和表格進行驗證

⚡ 核心功能

✅

事實聲明驗證

  • 檢查每個事實聲明是否有引用
  • 驗證引用來源確實存在
  • 確保引用與聲明一致
🔢

數值精確驗證

  • 96.555% ≠ 97%
  • 學術精確模式
  • 不接受四捨五入的數值
📊

支援多種格式

  • 讀取簡報 (PPT)
  • 解析 PDF 文件
  • 分析圖表與表格
🔄

雙重架構

  • 第一遍：擷取所有聲明
  • 第二遍：驗證每個聲明
  • 確保結果一致性

🔄 運作流程

LLM 輸出 → 擷取聲明 → 驗證引用 → 通過 → 發布
流程說明：系統採用兩次通過架構——先提取所有聲明，然後驗證。不再有不一致的結果。

關鍵創新

聲明提取規則

明確的分類法——統計數據、比較、歸屬等

狀態決策樹

確定性分類：已驗證 → 數值錯誤 → 幻覺

強制搜索模板

運行所有查詢模式，而非臨時搜索

🎯 兩種驗證模式

🌐

網路搜索驗證

  • 搜尋網路權威來源
  • 驗證引用確實存在
  • 檢查來源是否陳述聲稱內容
📄

文件僅驗證

  • 用戶提供來源文檔
  • 所有內容必須可追溯到這些文檔
  • 將任何外部知識標記為幻覺
觸發文件模式：「僅使用此文檔」/「僅驗證 PDF」

📊 驗證狀態說明

狀態說明
✅ Verified 與來源完全匹配
⚠️ Numerical Error 數值不匹配 (例：97% vs 96.555%)
⚠️ Unverified 找不到權威來源
❌ Hallucination 與來源矛盾或捏造
❌ Misleading 斷章取義或缺少脈絡
❌ Citation Not Found 引用的論文/報告不存在
❌ Not in Source 聲明無法追溯到提供的文檔（文件模式）
處理方式：如果檢查失敗，輸出應該被阻止或重新生成。
